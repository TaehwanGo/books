# Part 2. 아키텍처 스타일

- 아키텍처 스타일 vs. 아키텍처 패턴
- 아키텍처 스타일은 (모놀리식 배포 또는 별도 배포된 서비스의 레이어 내부에서 그렇듯) `유저 인터페이스와 백엔드 소스 코드를 구성하는 방법`, 그리고 `소스 코드가 데이터 저장소와 상호작용하는 방법`에 관한 결정적으로 중요한 `구조`입니다
- 아키텍처 패턴은 아키텍처 스타일에서 (일련의 작업 또는 서비스 간의 고확장성/고성능을 실현하는 방법처럼) 특정한 솔루션을 마련하는 데 유용한 `저수준의 설계 구조`를 말합니다
- 각각에 내재된 트레이드오프를 이해하자
